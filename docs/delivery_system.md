# 9MindTech Delivery System

This document defines the standard process used to onboard, deliver, and scale QA automation services for clients.

---

# Phase 1: Discovery

## Objective
Understand the client’s application, risks, and testing gaps.

## Checklist
- Understand product (what it does)
- Identify critical user flows
- Identify current testing method
- Identify release frequency
- Identify major pain points
- Confirm tech stack (web, API, GitHub, etc.)

## Output
- Clear understanding of client needs
- Initial test coverage ideas

---

# Phase 2: Proposal

## Objective
Provide a clear, simple solution with defined scope and pricing.

## Deliverables
- Automation framework setup
- UI tests (3–5 core flows)
- API tests (1–3 endpoints)
- CI pipeline (GitHub Actions)
- Reporting (HTML / Allure)

## Pricing Model
- One-time setup OR
- Monthly subscription

## Output
- Proposal document
- Service agreement
- Invoice (deposit)

---

# Phase 3: Onboarding

## Objective
Collect everything needed to begin development.

## Client Requirements
- Application URL (staging/test)
- Test credentials
- GitHub repo access
- API documentation
- Priority workflows

## Internal Setup
- Create client folder
- Define environment variables
- Confirm scope

---

# Phase 4: Execution

## Objective
Build and validate automation framework.

## Step-by-Step

### Day 1–2
- Review app
- Identify selectors
- Confirm test flows

### Day 3–4
- Build UI tests
- Build API tests

### Day 5–6
- Add CI pipeline
- Validate execution
- Fix instability

### Day 7
- Final validation
- Generate reports
- Clean documentation

---

# Phase 5: Delivery

## Objective
Provide a complete, usable testing solution.

## Deliverables
- Source code
- README
- CI pipeline
- Reports
- Documentation

## Handoff
- Walkthrough call
- Explain how to run tests
- Explain how CI works
- Explain reports and failures

---

# Phase 6: Completion

## Objective
Close project and collect final payment.

## Steps
- Send final invoice
- Confirm delivery acceptance
- Provide final files

---

# Phase 7: Retainer (Upsell)

## Objective
Convert one-time client into recurring revenue.

## Monthly Services
- Maintain tests
- Add new test coverage
- Support releases
- Fix failing tests

## Pricing
- $750–$1500/month

---

# Internal Rules

- No work starts without deposit
- Always define scope clearly
- Always document delivery
- Always offer retainer
- Always ask for testimonial
